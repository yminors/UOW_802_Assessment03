Common Web Attacks

Name Web Attacks
Cross-Site Scripting (XSS)


Source: https://www.invicti.com/blog/web-security/session-hijacking/

Cross-Site Scripting (XSS), also referred to as XSS attacks, empowers an attacker to run arbitrary JavaScript code within the browser of an unsuspecting victim user.

https://www.trendmicro.com/en_au/devops/23/e/cross-site-scripting-xss-attacks.html
Phishing


Source:https://learning.mlytics.com/cyber-attacks/what-is-a-phishing-attack/

A social engineering attack, where an attacker sends emails pretending to come from a trusted source.

https://www.ibm.com/topics/phishing/
Adware


Source:https://www.zenarmor.com/docs/network-security-tutorials/what-is-adware

Adware tracks a user's browsing activity to analyze behavior patterns and interests to enable advertisers to send the user targeted advertising.

https://www.imperva.com
SQL Injection


Source:https://portswigger.net/web-security/sql-injection

SQL injection is a technique employed by attackers to implant harmful code into a server that relies on SQL, to divulge information it would typically keep confidential.

https://www.cisco.com
Cross-Site Request Forgery (CSRF)


Source:https://www.zenarmor.com/docs/network-security-tutorials/what-is-adware

Cross-Site Request Forgery (CSRF), also known as session riding or one-click attack, enables an attacker to manipulate a victim user into unknowingly performing actions they did not intend to initiates.

https://www.cyber.gov.au/
Distributed Denial of Service (DDoS)


Source:https://www.fortinet.com/resources/

Distributed Denial of Servic attack involves overwhelming a target system,large volume of traffic, from multiple sources (botnets). The aim is to exhaust resources and bandwidth, causing the targeted system to become slow or completely unavailable to legitimate users.

https://www.f5.com/labs/learning-center/what-is-a-distributed-denial-of-service-attack/
Man-in-the-Middle (MIM) Attack


Source:https://www.geeksforgeeks.org/how-to-prevent-man-in-the-middle-attack/

Attackers intercept and alter communication between two parties. By doing so, the attacker can create the illusion of a normal information exchange taking place while they secretly monitor or manipulate the communication.

https://www.geeksforgeeks.org/how-to-prevent-man-in-the-middle-attack/ Source:
DNS Spoofing


Source:https://www.cloudns.net/blog/dns-spoofing-dns-poisoning/

DNS spoofing, also known as DNS cache poisoning, is an attack that involves manipulating the DNS resolution process. In this attack, the attacker sends false DNS responses to redirect users to malicious websites or incorrect IP addresses. It's a method of tricking users' devices i nto connecting to the wrong destination, potentially leading to data theft or other malicious actions.

https://www.cloudns.net/blog/dns-spoofing-dns-poisoning/
Brute Force Attack


Source:https://www.bankinfosecurity.com/blogs/credential-stuffing-attacks-vs-brute-force-attacks-p-2767

A brute force attack is a hacking technique that involves systematically attempting all possible combinations of passwords, login credentials, or encryption keys in order to gain unauthorized access.

https://www.fortinet.com/resources/cyberglossary/brute-force-attack
Remote Code Execution (RCE)


Source:https://beaglesecurity.com/blog/vulnerability/remote-code-execution.html 		Attackers exploit vulnerabilities to execute code on a remote server
